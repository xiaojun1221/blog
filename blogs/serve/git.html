                # 一、Git基础概念

                # 一些名词：VCS、CVS、SVN、Git、GitHub、GitLab

                • VCS (Version Control System)：版本控制系统的统称，CVS、SVN、GIT都可以称为VCS。
                • CVS (Concurrent Versions System)：一种集中式版本控制系统，后来逐渐被SVN替代。
                • SVN (Subversion)：一种集中式版本控制系统，仅源代码管理来说，基本已经被Git取代，但因为其强大的细粒度权限控制要强于Git，所以仍有一席之地。
                • Git：一种分布式版本控制系统，目前应用最广泛的源代码版本控制系统。
                • GitHub：一个面向开源及私有软件项目托管平台https://github.com，用于管理Git仓库，被微软收购
                • GitLab：开源和GitHub功能类似的托管平台(可以理解是GitHub的私服版)，类似于(Maven Repository与Nexus的关系)

                # 二、Git合并请求(Merge Request)

                • 说明：一种保护重要分支的方式
                • 原理：重要分支不允许直接推送，需要发起合并请求，由分支负责人审核通过以后合并上去

                • 几种分支的作用
                  • 特性分支(lwc_version)：某个版本的个人分支，用于将个人代码备份到云端，保护代码安全及不同设备的同步
                  • 开发分支(develop_version)：某个版本的开发分支，由负责人尽可能保证代码是可运行的分支，同时也是修复bug的分支，通常与test_version同步,通常发起提测以后，该版本的分支develop及特性分支，不允许再开发新需求，仅进入bug修复阶段，如果有新需求，需要用新版本号建立出一套分支进行迭代
                  • 测试分支(test_version)：某个版本的测试分支，通常在提测时，从develop_version合并过去

                # 三、基于IDEA的合并请求操作(IDEA版本：2020.1.2)

                • # 1.安装插件《GitLab Projects 2020》

                • 1. 配置git服务器:Settings > GitLab > Add New GitLab Server

                • 2. 设置参数及获取token

                • 3.获得token进行设置

                • 4.添加完成

                # 2.发起合并请求

                • 操作：VCS > Git > GitLab > Create Merge Request
                  • To Branch：要合并到的分支 > develop
                  • Assignee: 审核人(由小组达成共识)
                  • Title: 请求合并的描述
                  • Description:详细描述

                # 3.负责人审核请求

                • 操作：VCS > Git > GitLab > List Merge Request

                • 查看提交详情

                • 确认无误后合并

                • 如何驳回请求：暂时插件无法驳回，需要在浏览器上操作

                # 四、基于浏览器的合并请求操作

                # 1.发起合并请求

                • 操作：打开项目 > 合并请求 > New merge request

                • 选好源分支和目标分支

                • 确认好源分支和目标分支及审核人,发起合并

                # 2.负责人审核请求

                • 操作：打开项目 > 合并请求

                • 查看及合并

                • 无论合并成功还是失败，都可以添加评论进行交流(不常用，一般直接面对面或语音沟通即可)

                # 五、如何配置保护分支

                • 操作： 仓库 > 分支 > 项目设置 > Protected Branches
