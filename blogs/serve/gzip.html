      使用gzip压缩数据传输

      jun 2020-04-16 15:31:25 gzipfeign

      # 使用gzip压缩数据传输，及替换feign的通信实现为httpclient

      # 启用springboot压缩，其在返回响应时，将数据压缩

      • 修改application-xxx.yml文件，在server代码段中，增加以下配置以开启SpringBoot的压缩
      server:
  #启用响应压缩
  compression:
    #启用压缩
    enabled: true
    mime-types: application/json,application/xml,text/html,text/xml,text/plain
	#最小响应数据的大小，当响应数据大于此值时才会压缩(单位B)
    min-response-size: 2048

      # 启用feign压缩及解压响应

      • 修改application-xxx.yml文件，在feign代码段，增加以下配置启用httpclient及并且使feign也使用gzip压缩
      feign:
  hystrix:
    #启用熔断器
    enabled: true
  #启用feign压缩
  compression:
    request:
      enabled: true
      #压缩的Mime类型
      mime-types: application/json,application/xml,text/html,text/xml,text/plain
      #最小请求数据的大小，当请求数据大于此值时才会压缩(单位B)
      min-request-size: 2048
    #解压响应
    response:
      enabled: true
      #使用gzip解压
      useGzipDecoder: true
  httpclient:
    #使用httpclient作为实现
    enabled: true

      • 在服务提供者打断点，观察request对象，如果有以下header，表示feign请求时进行了压缩

      • 配置nginx,修改nginx.conf文件，以开启gzip压缩，该配置位于http代码段中
      http{
    gzip on;
    gzip_static on;
    gzip_min_length  1024;
    gzip_buffers     4 16k;
    gzip_http_version 1.1;
    gzip_comp_level 2;
    gzip_types     text/plain application/javascript application/x-javascript text/javascript text/css application/xml;
    gzip_vary on;
    gzip_proxied   expired no-cache no-store private auth;
    gzip_disable   &quot;MSIE [1-6]\.&quot;;
}
